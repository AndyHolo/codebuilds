 Platforms

The primary focus of this site is to provide builds of the open-source edition of VS Code for less common platforms (those that might not otherwise receive them).

 Architectures
i386, amd64, armhf, arm64
 Packages
DEB, RPM, Snappy, Flatpak, AppImage
 Operating Systems
Android*, Chrome OS*, Linux (* uses embedded Linux)

 Testing

For each and every release, the build system prepares a virtual system for the given architecture. The finished package is then installed and tested with the full suite of tests from Microsoft. Once this is done, a test report is prepared that can be downloaded for review. Only packages that pass all tests will proceed to release.

 Installation Instructions

 Scripted Linux Installation

(including Raspberry Pis, Odroids and other single-board computers)

Packages are currently available in DEB and RPM format for ARM, x86 and amd64 systems.

You can use the scripts here to install the packages and add their repository to your system.

Open a new terminal. If you need super-user rights (you probably do), then you can enter sudo -s and press return to enter a super-user session. Run the installer for your current distribution:

 APT instructions

(including Debian, Raspbian, Ubuntu and Linux Mint) 
. <( wget -O - https://codebuilds.org/installers/apt.sh )

 YUM instructions

(including Fedora, Pidora, Red Hat and SUSE) 
. <( wget -O - https://codebuilds.org/installers/yum.sh )

Press the return key. Once the installer has completed, you should have a "Code" entry in your desktop program list.

 Manual Linux installation

If for any reason the script above will not work on your system, or you do not want to add the package source for updates, you can get the latest version of the package for your system below.

 Public GPG key

For either APT or YUM installation, you'll want the public GPG key to verify the package, which you can download here.

 Frequently asked questions

What is this?
These are automated builds of the open-source edition of Microsoft's Visual Studio Code.
How does this work?
The latest code is pulled on a nightly basis from Microsoft's official repository. If there are updates, the code is built, tested, packaged and deployed.
Which devices are supported?
Please refer to the Platforms section.
Do these releases support extensions?
The extensions you want should work, so long as they do not depend on underlying tools that are not supported on your architecture.
Can I really run Visual Studio Code on my phone?
You may be able to run it, but I wouldn't recommend it unless you have a physical keyboard. The Android instructions are really intended for convertible tablets and docked handsets.
Do I need to "root" my Android device?
Absolutely not, and there are good security reasons not to.
Do I need to put my Chromebook into Developer Mode?
Absolutely not, and there are good security reasons not to.
Will Visual Studio Code run on my device?
The best way to find out if your device is compatible is to go to the Instructions section and try it for yourself.
Which virtual systems are these releases tested on?
32-bit ARM hard-float (armhf)
These packages are tested on a Raspbian Jessie image to mimic a Raspberry Pi 2.
64-bit ARM hard-float (arm64)
These packages are tested on a modified Ubuntu 16.04 image to mimic a Odroid C2.
Are these releases provided by Microsoft?
No. These releases are an independent community effort. For clarity, this website (and the releases here) are not affiliated with Microsoft.
Why is the logo so much different from the one on Microsoft's website?
The Visual Studio Code logo is owned by Microsoft, and is reserved for use on releases provided directly by them.
Where are the packages being hosted?
The release packages are hosted by JFrog Bintray, who are graciously providing this service for free under their open-source plan.
This is really cool. Can I help with this?
Yes you can! Any and all help is appreciated, but you'll need to digitally sign the Microsoft CLA first, so that we can contribute any fixes back upstream to the core repository.
So who maintains this site?
I'm Jay. Nice to meet you. You can find my details in the contact section.
Are you a Microsoft MVP?
I am not.
